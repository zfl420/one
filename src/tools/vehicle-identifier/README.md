# 车型识别工具

通过VIN码（车辆识别代码）识别车型信息的工具，支持图片OCR识别和手动输入两种方式。

## 功能特性

- ✅ **双重输入方式**
  - 图片上传：支持拖拽和点击上传VIN码图片进行OCR识别
  - 手动输入：直接输入17位VIN码查询车型信息

- ✅ **详细信息展示**
  - 品牌、车型、年款、厂商等基本信息
  - 发动机型号、排量、变速箱类型等配置信息
  - 车身结构、驱动方式、燃料类型等参数

- ✅ **历史记录**
  - 自动保存最近20条查询记录
  - 支持查看和删除历史记录
  - 手动输入的VIN码可快速重新查询

- ✅ **友好交互**
  - 实时输入验证
  - 加载状态提示
  - 错误信息提示
  - 响应式设计，支持移动端

## 使用说明

### 图片识别方式

1. 切换到"图片识别"标签
2. 点击上传区域选择图片，或直接拖拽图片到上传区域
3. 确认预览图片无误后，点击"开始识别"按钮
4. 等待识别完成，查看车型信息

**图片要求：**
- 格式：JPG、PNG、BMP
- 大小：不超过4MB
- 建议：VIN码清晰可见，高清图片识别率更高

### 手动输入方式

1. 切换到"手动输入"标签
2. 在输入框中输入17位VIN码
3. 系统会自动验证格式并显示字符计数
4. 点击"查询车型"按钮
5. 等待查询完成，查看车型信息

**VIN码说明：**
- 必须是17位字符
- 不包含字母 I、O、Q（易与数字混淆）
- 不区分大小写，系统会自动转为大写

### VIN码查找位置

- 行驶证上的车架号
- 挡风玻璃左下角（从车外看）
- 发动机舱铭牌
- 车门B柱或门框上的标签

## 技术实现

### API集成

使用 17VIN API 进行车型识别：
- 图片识别接口：`vin_ocr_and_vin_decode`
- VIN码查询接口：`vin_decode`

### 状态管理

使用 Zustand 管理应用状态：
- 输入模式切换
- 识别结果存储
- 历史记录管理
- 加载和错误状态

### 组件结构

```
VehicleIdentifier.tsx (主组件)
├── ImageUpload.tsx (图片上传组件)
├── VinInput.tsx (VIN码输入组件)
├── ResultDisplay.tsx (结果展示组件)
└── HistoryPanel.tsx (历史记录组件)
```

## 环境配置

需要在 `.env.local` 文件中配置API凭证：

```env
VITE_17VIN_USERNAME=your_username
VITE_17VIN_PASSWORD=your_password
VITE_17VIN_API_URL=http://api.17vin.com:8080/
```

## 依赖项

- `crypto-js`: MD5加密生成API token
- `zustand`: 状态管理
- `react`: UI框架

## 注意事项

1. **API限制**
   - 需要有效的API账号和密码
   - 注意API调用频率限制
   - 图片大小不超过4MB

2. **隐私保护**
   - 历史记录仅保存在本地浏览器
   - 不上传任何个人敏感信息
   - 可随时清空历史记录

3. **识别准确率**
   - 图片质量直接影响识别准确率
   - 建议使用高清、光线充足的图片
   - VIN码应完整清晰可见
